Form Field Text Input met Value Type

In dit voorbeeld gebruikt het formulier geen String, maar het domeintype PersonFullName. Dit value type vertegenwoordigt de volledige naam van een persoon en bevat zelf de validatieregels.

In het PersonFullName type worden bijvoorbeeld constraints afgedwongen zoals een minimale en maximale lengte van de naam. Hierdoor kan een ongeldige naam niet in het domeinmodel terechtkomen.

Formulieren werken in de browser altijd met tekst (String). Daarom gebruikt Wicket een IConverter om de invoer uit het formulier te converteren naar het value type PersonFullName. In deze converter worden eventuele validatiefouten vertaald naar gebruikersvriendelijke foutmeldingen.

Het gebruik van value types kost iets meer implementatietijd (bijvoorbeeld converters en integratie met frameworks), maar levert op lange termijn duidelijke voordelen op:

  • betere type safety in het domeinmodel
  • validatie op één centrale plek
  • duidelijkere betekenis van waarden in de code
  • minder kans op bugs door verkeerd gebruik van String

In dit voorbeeld wordt de tekstinvoer uit het formulier automatisch geconverteerd naar een PersonFullName en opgeslagen in het formuliermodel.
